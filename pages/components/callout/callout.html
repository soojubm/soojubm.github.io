  • ( + 게임에서 파티 탈퇴와 같이 직접 실행하지 않은, 시간을 특정하기 힘든 연속적인... 경우 여러 개가 필요하구나.)
  • 아이콘으로 callout의 역할을 구분할 수 있다. 따라서 아이콘이 없는 info 형태의 callout은 공허한 컴포넌트다. title / message

    • toast=>snackbar(withAction) - 정리하기, single line / multiple lines (head,body 텍스트)

  • 역할로 구분하면 input feedback은 callout과 분리해야 한다.
  • 닫기 또는 되돌리기 등 액션을 포함하지 마세요. 사용자 인터렉션이 요구되는 경우 주목도가 높은 모달로 제공하세요.
  • 사용자의 귀책이 아닌 경우, 단순 "오류" 대신 "시스템 오류"와 같은 용어를 사용하여 사용자가 잘못한 것처럼 느끼지 않도록 하세요.
  • 브라우저가 role="alert" 속성이 있는 요소를 발견하면 즉시 사용자에게 경고 내용을 읽습니다.
