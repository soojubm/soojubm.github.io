Foundations Signifiers define the visual cues that communicate what an element means, what state it is in, and what users can do with it. 기표는 사용자가 UI를 보고 해당 요소의 역할, 상태, 가능한 행동을 이해할 수 있도록 하는 시각적 단서입니다. 이 페이지는 디자인 시스템 안에서 특정 시각 표현이 특정 의미에만 사용되도록 제한하는 공통 계약을 정의합니다.

Overview

Signifier 페이지의 목적은 UI의 시각 표현이 어떤 의미, 상태, 행동 가능성을 나타내는지 일관되게 정의하는 것입니다. 기표는 장식이 아니라 사용자가 인터페이스를 해석하는 방식에 직접 관여하는 규칙입니다. 예를 들어 primary color text, underline, pointer cursor, external link icon, check icon은 각각 특정한 의미를 갖습니다. 이러한 단서가 다른 의미로 재사용되면 사용자는 요소의 역할을 오해할 수 있습니다.

Core Principles

One signifier, one primary meaning

하나의 시각적 단서는 시스템이 명시적으로 허용하지 않는 한 다른 의미로 재사용하지 않습니다. 예를 들어 외부 링크 아이콘은 외부 목적지로 이동한다는 의미에 사용하며, 일반적인 내부 이동이나 상세 보기에는 사용하지 않습니다.

Do not use interaction cues as decoration

클릭 가능성을 암시하는 색상, 밑줄, 커서, 아이콘은 장식이나 일반 강조로 사용하지 않습니다. 사용자는 이러한 단서를 인터랙션 가능성으로 학습하기 때문입니다.

Do not rely on color alone

상태나 피드백은 색상만으로 전달하지 않습니다. 중요한 상태는 아이콘, 텍스트, 형태, 위치, ARIA 속성 등과 함께 제공해야 합니다.

Keep signifiers consistent across components

컴포넌트의 접근성 속성은 다를 수 있지만 사용자가 인지하는 시각적 기표는 일관되게 유지합니다. 예를 들어 Tabs는 aria-selected, Toggle Button은 aria-pressed, Checkbox는 aria-checked를 사용할 수 있지만 선택 상태의 시각적 단서는 공통된 원칙을 따릅니다.
Clickable & Link Signifiers Clickable signifier는 사용자가 특정 요소를 누르거나 이동할 수 있음을 나타냅니다. primary color text, underline, pointer cursor, button-like shape은 인터랙티브 요소에만 사용합니다.

Primary color text

Primary color text는 링크, 텍스트 버튼, 인라인 액션처럼 클릭 가능한 텍스트에 예약합니다. 본문 강조, 메타 정보 강조, 장식적 하이라이트에는 사용하지 않습니다.

인터렉션 가능한 오브젝트를 구분합니다. 사용자는 클릭할 수 있는 오브젝트를 명확하게 인지하여 예측 가능한 신뢰를 구축할 수 있습니다. elevated 오브젝트는 클릭할 수 있다. 또는 오브젝트 내부에 클릭할 수 있는 요소가 존재한다. 연한 회색 배경의 오브젝트는 클릭할 수 있다. 브랜드 컬러 텍스트는 클릭할 수 있다. 이미지는 클릭할 수 있다?
