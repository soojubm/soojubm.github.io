Signifier 페이지의 목적은 UI의 시각 표현이 어떤 의미, 상태, 행동 가능성을 나타내는지 일관되게 정의하는 것입니다. 기표는 장식이 아니라 사용자가 인터페이스를 해석하는 방식에 직접 관여하는 규칙입니다. 예를 들어 primary color text, underline, pointer cursor, external link icon, check icon은 각각 특정한 의미를 갖습니다. 이러한 단서가 다른 의미로 재사용되면 사용자는 요소의 역할을 오해할 수 있습니다.

Page Context Contrast

페이지의 맥락을 구분하기 위해 컨테이너 너비, 레이아웃 모듈, 배경색의 대비를 사용할 수 있습니다. 이러한 대비는 사용자가 현재 화면의 성격을 빠르게 이해하도록 돕는 페이지 레벨의 기표입니다.

예를 들어 비밀번호 찾기와 같은 집중형 작업은 좁은 컨테이너를 사용하고, 로그인/회원가입처럼 더 많은 입력과 선택이 필요한 화면은 상대적으로 넓은 컨테이너를 사용할 수 있습니다. 설정, 글쓰기, 소개 페이지처럼 서로 다른 정보 구조를 가진 화면은 배경색이나 표면 대비를 통해 맥락의 전환을 표현할 수 있습니다.

Feedback Signifiers Feedback signifier는 사용자의 행동이나 시스템 상태에 대한 결과를 전달합니다. success, info, warning, error는 색상뿐 아니라 명시적 메시지와 접근성 관계를 함께 제공해야 합니다. Interaction states communicate the status of an element in the interface, establish confidence once an action is taken, and suggest the ability (or inability) to interact with the element. 에러 피드백은 사용자가 문제의 원인과 영향을 받는 대상을 명확히 이해할 수 있도록 제공해야 합니다. 색상만으로 에러 상태를 전달하지 말고, 에러 톤, 명시적인 메시지, 영향을 받는 필드 또는 영역, 그리고 접근 가능한 관계를 함께 제공해야 합니다. 폼 컨트롤의 경우 aria-invalid="true"로 유효하지 않은 상태를 표시하고, aria-describedby를 사용해 에러 메시지와 컨트롤을 연결합니다.
