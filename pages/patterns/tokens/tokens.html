Notes
  • https://www.delldesignsystem.com/foundations/typography/
  • 콘텐츠 컨테이너의 가로 크기. custom media의 max-width를 layout + @로 정의.
  • 폰트가 변경되면 폰트마다 baseline이 달라서 텍스트의 상하 간격이 안 맞을 수 있다. 텍스트노드(레이블 역할)의 패딩용 변수를 활용.
  • animation :: 콘텐츠보다 액션이 강조되지 않도록 뷰포트 내에서 delay를 최대 3개로 제한합니다. inactive로 변경될 때(닫기 등) 즉시(instantly) 실행 합니다.
  • display grid를 사용하면 마진과 거터를 제외한 나머지가 결국 칼럼이 됨. 또는 width 시스템으로 대체할 수 있다.
  • 그림자를 이용한 시각적 상승 효과로 주변 요소 대비 강조. 더 높은 우선순위 및 행동유도를 위한 높은 고도. 사용자는 시각적 단어로 우선순위를 인식하고 인터렉션. 시각적 우선순위는 그림자로만 줄 수 있는 것은 아님. overay 등으로 우선순위를 설정할 수도 있음.
  • 개별 토큰에 디스크립션이 필요하지 않도록 명명합니다. core: give a name to a raw value to enable theming and make improve maintainability
  • 컴포넌트 토큰은 상대적으로 변경 가능성이 높고 범용성이 낮기 때문에 컴포넌트 스코프에서만 정의. 컴포넌트 토큰을 정의하면 형태 기반의 컴포넌트를 정의 최소화할 수 있다.
  • --[namespace]-[component / category]-[attribute]-[status] 프로덕션이 아니기 때문에 namespace를 생략.
  • theme. 인버트 컬러셋을 추가로 정의하지 않습니다. 하나의 컬러셋으로 다크 테마에 대응합니다. 어렵고 또는 불필요한 규칙. 왜냐하면 초기 토큰 세팅 허들을 넘으면 컴포넌트 토큰을 활용한 테마 대응이 간편해짐.
