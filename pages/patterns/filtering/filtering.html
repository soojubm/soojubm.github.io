필터의 적용
  • 사용자는 적용된 필터 옵션을 초기화하기 위해 새로고침을 할까?
  • range UI는 얼마나 사용자 친화적인가?
  • presentation 컴포넌트가 필터의 적용과 상태에 영향을 준다.
  • 옵션 목록을 나열할 때. 모두 셀렉트로 제공할 수도 있지만 다른 컴포넌트로 옵션 하나 하나를 환기할 수 있다. dropdown, chip, radio 등의 혼합 => 디자인과 개발 비용이 증가하는 만큼 사용자 경험이 월등한가? (case. airbnb) 아니면 다른 의도나 효과가 있을까?
  • FilterActions. 필터를 적용하고 초기화하는 버튼 그룹의 정렬.
  • 모든 필터 옵션을 세로 목록형 가로 스크롤로 나열할 수도 있다. 큰 화면에서 유용하다. (case. bilibili, 관리자용)
  • 적용된 필터 상태를 removable chips로 추가 제공(case. 오늘의집, 네이버카페)하는 것은 효용이 있을까? => 없다! 공간 낭비. 리소스 낭비.
  • 필터링와 소팅의 위계. 소팅은 필터링의 하위 위계임을 시각적으로(위치 등) 명시. 또는 '정렬' 텍스트 레이블링으로 구분
