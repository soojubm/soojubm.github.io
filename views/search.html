Search Pattern

글로벌 검색 패턴과 로컬 검색 패턴의 구분. 검색의 인지.

  • 검색 결과 쿼리를 몇 개 제공할 것인가. 모바일 키보드 고려.
  • iOS의 검색 패턴 따로 정의하기. onFocus.
  • 검색 결과 페이지의 공유를 허용할 것인가?

검색의 시작

  • 해시태그나 카테고리 등 서비스가 제공하는 인기/추천 키워드의 클릭을 통한 검색. (액션/button) - clickable과 non-clickable 요소를 함께 정의하면 좋다.
  • 직접 키워드 입력 - 검색어 자동완성 (입력/input)

최근 검색 내역의 삭제

  • 개별 검색 내역의 삭제.
  • 전체 검색 내역의 삭제. 사용자에게 유용할 수도 있지만 리스크를 고려하라. 서비스의 특성에 맞게 제공.

검색의 실행

  • onChange: 키보드를 입력할 때 마다.
  • onSubmit: 검색 버튼을 클릭 또는 enter key를 입력.

검색의 결과

  • 검색어와 일치하지 않더라도 유사한 검색 결과를 항상 하나 이상 보여줄 수 있다. onSubmit에는 해당하지 않는다.
  • 상태(결과 없음)를 보여줄 수 있다. 또는, 추천 검색어나 추천 콘텐츠를 보여줄 수 있다. 추천 콘텐츠를 노출하는 것은 ott 등의 컨텐츠 플랫폼에서 유용할 수 있지만 명확한 결과를 기대할 때는 주의.
  • 결과의 필터링 또는 소팅. link to filtering pattern page
