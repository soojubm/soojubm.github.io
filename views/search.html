검색의 인지

  • 키워드 입력
  • 해시태그나 카테고리 등 서비스가 제공하는 인기/추천 키워드의 클릭을 통한 검색. (액션/button)
  • 검색필드가 focusable이 아닌 clickable 액션인 케이스. (readonly) 검색탭으로 이동하여 콘텐츠 탐색과 검색.

검색의 시작
onFocus - Searchfield

  • tabIndex: back - input
  • onBlur. (default) 인기 콘텐츠 또는 추천 콘텐츠 등 개인화 콘텐츠 탐색할 수 있다.
  • onFocus. 검색내역을 표시하고 검색 키워드를 입력할 수 있다. 취소 버튼으로 onBlur.

검색의 시작, 최근 검색 내역
onFocus - value.length === 0

  • 개별 내역의 삭제. 휘발되어도 치명적이지 않은 정보라 편집 모듈 없이 삭제 버튼으로 제공한다.
  • 전체 내역의 삭제. 사용자에게 유용할 수도 있지만 리스크를 고려. 서비스의 특성에 맞게 제공. 검색 내역의 삭제가 개인화 콘텐츠에 영향을 미치는가.

검색의 실행
onChange - 키보드를 입력할 때 마다

  • debouncing.
  • 검색어의 자동완성 또는 검색의 기대 결과를 미리 보여준다.
  • 결과 없음(상태)을 표시하기 보담 일치하는 결과가 없더라도 유사 콘텐츠 또는 추천 콘텐츠로 대체하여 하나 이상의 결과를 항상 보여줍니다. 큐레이션이 강화된 콘텐츠 플랫폼에서 이탈 방지를 위해 활용할 수 있지만 명확한 결과를 기대할 때 오히려 좋지 않은 경험이 될 수 있다.

검색의 실행
onSubmit - 검색 버튼을 클릭 또는 enter key

  • 결과 없음(상태)을 표시할 수 있다.
  • 하나 또는 하나 이상의 명확한 결과가 필요한 경우.
  • 개발 리소스 적다.

검색의 결과

  • 목록. 필터링 또는 소팅. 모바일 키보드의 높이가 고려된 쿼리의 갯수
  • 결과 페이지의 공유를 허용할 것인가?
