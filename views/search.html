검색의 준비

최근 검색 내역
onFocus - value.length === 0

  • 개별 내역의 삭제. 휘발되어도 치명적이지 않은 정보라 편집 모듈 없이 삭제 버튼으로 제공한다.
  • 전체 내역의 삭제. 사용자에게 유용할 수도 있지만 리스크를 고려. 서비스의 특성에 맞게 제공. 검색 내역의 삭제가 개인화 콘텐츠에 영향을 미치는가.
  • 검색 내역을 삭제하는 것과 검색 히스토리를 삭제하는 것은 구분해야 한다.
  • 검색 내역을 로컬에 저장할 것인가?

검색의 실행

검색어 추천 및 자동완성.
onFocus - value.length > 0 / onChange - 키보드를 입력할 때 마다

  • 결과 없음(상태)을 표시하지 않습니다. 일치하는 결과가 없더라도 유사 콘텐츠 또는 추천 콘텐츠로 대체하여 하나 이상의 결과를 항상 보여주어야 합니다. 큐레이션이 강화된 콘텐츠 플랫폼에서 이탈 방지를 위해 활용할 수 있지만 사용자가 명확한 결과를 기대하는 경우 좋지 않은 경험이 될 수 있다.
  • debouncing.
  • 검색어의 자동완성하고 검색의 결과를 키워드를 입력할 때마다 갱신한다.

onSubmit - 검색 버튼을 클릭 또는 enter key

  • 결과 없음(상태)을 표시한다. 하나 또는 하나 이상의 명확한 결과를 요구하는 경우. 체류시간을 확보해야 하는 콘텐츠 플랫폼에는 적합하지 않다.
