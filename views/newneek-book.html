뉴닉 고슴책

아이디어를 검증을 위해 해당 피쳐의 프로세스 위주로 설명

문제 2 : 뉴스레터에서 소비된 콘텐츠를 웹사이트에서 재게시하기 때문에 독자적인 콘텐츠 부족. 구독자의 방문율이 떨어지고 방문 근거가 부족.

솔루션 : 어떻게 하면 추가적은 콘텐츠를 생산하지 않고 새로운 콘텐츠를 소비하도록 할 수 있을까? 신청 도서와 관련된 뉴닉 콘텐츠를 함께 제공.

정적 페이지로 만들고 구글 폼으로 신청 받음. 수천건의 신청 발생.

문제: 사용자들이 책을 신청할 때마다 구글 폼에서 배송지를 반복해서 입력해야 한다.

솔루션: 자체 신청폼을 구축하여 입력한 정보를 저장해준다.

에디터가 백오피스에서 1~2주 마다 새로운 책을 등록하고 마감날짜를 지정할 수 있는 시스템 구축

신청 가능한 책과 마감된 책 필터링 기능 제공

  1. 배송지 저장. 반복적으로 정보를 입력해야 하는 경험을 개선하고, 예정되어 있던 굿즈 구매 과정에서 배송지 입력 절차를 간소화..
  2. 매번 1000~3000명의 구독자가 가입 전환. 가입 전환 핵심 지표는 아니었음.
  3. 관심분야, 성별 사용자 데이터를 수집 => 신규 콘텐츠 개발에 참고

수집한 데이터의 한계 :: 뉴니커 일반이 아닌 독서를 좋아하는 사용자에 한정된 데이터
