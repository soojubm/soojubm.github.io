Layouts

페이지 레벨과 섹션 레벨의 레이아웃을 정의합니다.

  • 섹션, 아티클 그리고 모달, 드롭다운 등의 컴포넌트도 작은 화면에서는 페이지 맥락이 될 수 있습니다.
  • 모바일 퍼스트, 가로 스크롤 트랜드 기조에서 12 column grid 등 엄격한 그리드 시스템을 정의하지 않습니다.

  • 컨테이너는 최대 4개 칼럼을 가집니다.

  • 리스트는 언제든 가로 스크롤로 변경할 수 있도록 설계합니다.
  • 페이지의 맥락에 따라 컨테이너 너비로 대비를 줄 수 있습니다. 예를 들면 비밀번호찾기와 로그인/회원가입 페이지는 각각 다른 레이아웃 모듈로써 역할(맥락)을 구분합니다.

  • 설정, 글쓰기, 소개 등 페이지에서 의도적으로 레이아웃 모듈이나 배경색으로 대비를 줄 수 있습니다.

"Flat" Site Architecture

사이트맵을 만들고 유지하는 이유는 무엇일까요?

필요 이상으로 사이트가 복잡해지지 않도록 하기 위해서에요.

A Flat Architecture means that users (and search engine crawlers) can reach any page on your site in 4 clicks or less.

URL Structure Category Pages
  1. 프로젝트에 참여하는 모든 사람이 액세스 할 수있는 형식으로 사이트 맵 유지
  2. 웹 사이트 구조에서 일어나는 모든 변화를 반영하는 살아있는 문서. 변경/ 반영이 쉬워야 한다.
  3. 멘탈 모델은 뭔가 현실 세계에서 작동하는 방법에 대한 다른 사람의 생각에 대한 설명입니다. Jakob의 법칙에 따르면 사용자는 대부분의 시간을 다른 사이트에서 보냅니다. 이는 사용자가 이미 알고있는 다른 모든 사이트와 동일한 방식으로 작동하는 사이트를 선호한다는 것을 의미합니다. 이 규칙은 시각 디자인과 정보 아키텍처 모두에 적용됩니다. 즉, 웹 사이트의 시각적 모양과 논리적 구조가 해당 카테고리의 경쟁 업체 웹 사이트와 크게 다르지 않아야합니다. 예를 들어 회사 웹 사이트를 디자인하는 경우 대부분의 방문자가 기대하는 정보는 비슷하기 때문에 "회사 소개"페이지를 포함하고 최상위 탐색 옵션으로 만드는 것이 좋습니다. 대상 청중을 대표하는 사람들과 함께 공개 카드 정렬 워크샵 을 수행하고 콘텐츠를 그룹으로 구성하고 그룹의 이름을 제시하도록하는 것이 좋습니다 .
  4. 사용자가 작업을 완료하기 위해 수행해야하는 단계가 많을수록 (예 : 웹 사이트에서 제품 구매) 방문자가 작업을 완료하지 않고 사이트를 떠날 가능성이 높아집니다. 3 번 클릭 규칙 은 웹 사이트의 모든 페이지를 탐색하는 데 3 번의 클릭 만하면된다는 비공식적 인 웹 디자인 규칙입니다. 이는 사이트 사용자가 좌절감을 느끼고 세 번의 클릭으로 정보를 찾을 수 없으면 종종 떠날 것이라는 믿음에 기반합니다. 사이트 맵을 사용하여 필수 단계와 선택 단계를 파악하고 가능한 경우 단계를 결합하세요.
