부끄러우니까 보지마요

구조와 표현, 행동을 엄격하게 구분합니다. 하나의 엘리먼트는 하나의 시멘틱한 클래스를 가집니다.

subject본문 들어가기

일관된 코드로 생산성을 높히기 위한 가이드를 작성합니다. 에어비앤비와 구글의 스타일시트/자바스크립트 가이드를 참고했습니다. 효율적인 코드를 위해 고민하고 기록하며 현재 제가 사용하는 방법론을 적용한 스타일 가이드를 만들어 봅니다.

이 프로젝트는 코드를 간소화하고 효율적인 유지보수를 위한 코드를 연구한다.

개요

  • 인덱스 페이지를 메인 페이지가 아닌 홈 페이지로 명명하여 메인 태그와 구분합니다.
  • 들여쓰기는 2칸의 공백을 가지는 탭 키를 사용합니다.
  • 아이디 선택자는 재사용과 오버라이드를 통한 재정의가 어렵기 때문에 사용하지 않습니다.
  • 바닐라 자바스크립트를 사용합니다.

웹 접근성Web Accessibility

  • ARIA 상태를 사용합니다. (ex 숨김 텍스트는 aria-hidden="true" 어트리뷰트를 사용하여 별도의 클래스를 사용하지 않습니다.)

HTML

  • 구조를 최대한 단순화해요. 단순화된 구조는 유지보수 과정에서 스타일시트의 변경만으로 유아이의 변화에 대응할 수 있어야 해요. 따라서 float이나 flex는 지양하고 position을 적극 활용해요.
  • 모든 엘리먼트는 하나의 시멘틱한 클래스만을 가져야 해요. 클래스 네이밍이 과도하게 길어질 수 있으니 아토믹한 요소는 네스팅하여 정의하는 것을 허용합니다.

스타일시트

케밥 케이스kebab-case를 사용해요.

속성 간 개행하지 않습니다. 띄어쓰기는 선택자와 대괄호 사이 1회 사용합니다. 개행하지 않는 것이 구조 파악에 훨씬 좋으며 속성의 색상 차만으로 가독성을 담보한다고 생각해요. 그리고 디자인되어 있다는 인상을 받습니다. 하지만 글로벌 트랜드는 개행하는 것이므로 여러분과 함께 하는 프로젝트에서는 개행하도록 하겠어요.

z-index는 한 자리수로 사용합니다. 예시 모달 8. 모달 위 버튼 9. 드로어 5.

마진과 패딩에 rem을 단위를 사용합니다.

자바스크립트

카멜케이스camelCase를 사용합니다.

const documentBody = document.body;
const bodySection = documentBody.querySelector('section');

스크립팅 클래스는 js- 접두어를 사용하여 구분하고 별도의 스타일은 적용하지 않습니다.

변수가 오브젝트라는 것을 알 수 있도록 Object를 사용한다. (??)

Object.values(data).forEach(() => {})

매거진이나 블로그에 사용되는 포스트 유아이입니다. 포스트를 구성하는 컴포넌트들을 배치하고, 비대칭적 레이아웃과 타이포그래피 이론까지 실습해봅니다.
