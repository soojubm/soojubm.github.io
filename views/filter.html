Presentation

  • Dropdown: desktop에서 상. moblie에서 중.
  • Sheet: desktop에서 하. mobile에서 상.
  • Modal: desktop과 mobile에서 중. desktop과 mobile에 동일한 경험과 낮은 개발 비용.

필터 옵션의 presentation 컴포넌트는 환경에 최적화된 경험을 위해 변경하기 쉬워야 한다.

presentation 컴포넌트가 필터의 상태나 적용 방법에 많은 영향을 준다.

필터의 적용

  • Instant updates. 각각의 필터를 즉시 적용 (onChange) - 더 좋은 사용자 경험 but 데이터 소모
  • Batch updates. 적용 버튼을 클릭할 때 적용 (onSubmit) - multiple filters

적용된 필터의 해제Reset

  • 부분 해제
  • 전체 해제

적용된 필터의 상태

  • Multiple Options: 빌터 버튼에 적용된 필터의 갯수 표시.
  • Single Option: 액티브 상태와 레이블을 표기. (todo multiple)
  • 적용된 필터 상태를 chip group with removable로 추가 제공(case. 오늘의집/네이버카페)

필터링와 소팅의 위계

같은 컴포넌트일지라도 위계는 다르다. 소팅은 필터링의 하위 위계임을 시각적으로(위치 등) 명시하라.
