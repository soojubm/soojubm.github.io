Presentation

  • Dropdown with chip group: desktop에서 상. moblie에서 중. desktop과 mobile에 동일한 경험과 낮은 개발 비용. 막연한 탐색.
  • Sheet: desktop에서 하. mobile에서 상.
  • Modal: desktop과 mobile에서 중. desktop과 mobile에 동일한 경험과 낮은 개발 비용.
  • presentation 컴포넌트가 필터의 상태나 적용 방법에 많은 영향을 준다. 필터 옵션의 presentation 컴포넌트는 환경에 최적화된 경험을 위해 변경하기 쉬워야 한다.

필터의 적용

  • Instant updates. 각각의 필터를 즉시 적용 (onChange) - 더 좋은 사용자 경험 but 데이터 소모
  • Batch updates. 적용 버튼을 클릭할 때 적용 (onSubmit) - 주로 multiple filters. single filter에서도 사용하는 케이스가 있음...

적용된 필터의 상태

  • Multiple Options: 필터 트리거 버튼에 적용된 필터의 갯수 표시.
  • Single Option: 액티브 상태와 레이블을 표기. (todo multiple)
  • 적용된 필터 상태를 removable chips로 추가 제공(case. 오늘의집, 네이버카페) => 이게 의미있는 인터페이스인가?

필터링와 소팅의 위계

같은 컴포넌트일지라도 위계는 다르다. 소팅은 필터링의 하위 위계임을 시각적으로(위치 등) 명시하라.

궁굼이

  • 사용자는 적용된 필터 옵션 전체를 초기화하기 위해 새로고침을 할까?
  • 하나의 화면에서 모든 옵션 목록을 나열할 때 옵션 별로 다른 컴포넌트로 제공할 수 있다. 예를 들면 드롭다운, chip, radio의 혼합 => 디자인과 개발 비용의 증가 만큼 사용자 경험이 월등한가? (case. airbnb)
  • 필터의 버튼 그룹의 정렬 기준을 정의. hero 섹션과 같고 dialog와는 다르다.
  • range option
  • 모든 필터 옵션을 세로 목록형 가로 스크롤로 나열할 수도 있다. 큰 화면에서 상대적으로 유용하다.
