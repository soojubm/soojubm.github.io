Buttons

과업 수행을 위한 페이지 이동, 데이터 제출 등 사용자 여정의 핵심적인 액션입니다.

  • 버튼의 상태. 상태를 표현하는 시각 언어는 일부 컴포넌트에 종속되지 않고 제품 전체에 일관되어야 합니다. 색각 이상자를 위해 색상 외의 시각 언어로 상태를 표현하세요.
  • 버튼의 실행. 액션 실행 후 피드백합니다. 물리적인 인터페이스의 출현 뿐 아니라 예측할 수 있는 페이지 이동이나 페이지 새로고침, 정보 구조의 변화도 피드백으로 취급될 수 있습니다. (bad case. 인스타그램의 로그인 성공 toast)
  • 버튼의 레이블. 사용자가 다음에 일어날 일을 예측할 수 있도록 텍스트를 작성하세요. 부득이 텍스트 레이블을 생략하는 경우 보편적인 메타포의 아이콘을 사용하고 대체 텍스트를 제공해야 합니다..
  • 버튼의 아래에 실행 유도나 실행 결과를 가이드하는 텍스트를 배치하지 마세요. (secondary 액션의 text 버튼이 아래에 위치하는 패턴 고려). 제목/디스크립션/버튼레이블도 충분.

    • text. 버튼은 버튼 처럼 보이지 않을 수 있기 때문에 특정 패턴에서만 사용하세요.

    large. 프로모션 페이지의 call to action으로 제한적으로 사용합니다.

    show more. 사용자가 정보를 탐색할 때 행동 유도성을 높이기 위해 단일 콘텐츠 또는 콘텐츠 목록보다 작은 크기로 대비를 주세요.

  • 작업 / 액션 / 실행

Editing checklist

  • 최대 8자
  • 사용자 시점 (ex 보내기 => 받기)
  • 레이블 생략(truncated) 불가능
  • 동사 포함

Accessibility

Related components

button group.
  • 최대 2가지 액션을 사용할 수 있습니다. 3가지 이상의 액션 모음을 위한 toolbar 컴포넌트를 정의하기.
  • 버튼그룹에서 primary 버튼과 secondary 버튼을 함께 배치하지 마세요.
  • 버튼그룹에서 항상 primary 버튼을 먼저 마크업하거나 tab index를 설정하여 탭으로 먼저 접근할 수 있도록 하세요.
