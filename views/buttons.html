Best practices

  • 액션 실행 후 피드백합니다. 물리적인 인터페이스의 출현 뿐 아니라 예측할 수 있는 페이지 이동이나 페이지 새로고침, 정보 구조의 변화도 피드백으로 취급될 수 있습니다. (bad case. 인스타그램의 로그인 성공 toast)
  • 사용자가 다음에 일어날 일을 예측할 수 있도록 레이블을 작성하세요. 텍스트 레이블을 생략하는 경우 보편적인 메타포의 아이콘과 대체 텍스트를 제공하세요.
  • 통일된 시각 언어로 버튼의 상태를 표현하세요. 상태를 표현하는 시각 언어는 일부 컴포넌트에 종속되지 않고 제품 전체에 일관되어야 합니다.

Editing checklist

  • The button text should clearly indicate what will happen when the button is clicked.
  • 최대 8자
  • 사용자 시점 (ex 보내기 => 받기)
  • 레이블 생략 불가능
