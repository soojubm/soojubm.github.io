Chips 🔥

사용자 여정을 보조하는 액션입니다. 실행 페이지 내에서 사용자가 즉각적인 결과를 기대할 수 있습니다.

a.k.a. Tag Pill Badge Token Label

Best practices

  • 맥락이 다른 chip을 나열할 수 있습니다. 이 때 단일선택, 다중선택, 삭제여부, 펼침메뉴 등의 기표를 충분히 제공하세요.
  • 콘텐츠 정렬sorting을 위한 옵션을 chip group으로 chip을 나열하지 마세요. dropdown으로 현재 정렬 상태만 요약합니다. (bad case: 2022.04 구글 지도 바뀐 것)

콘텐츠 필터링을 위한 칩 그룹에서, 액티브 상태에 브랜드 컬러를 쓰지 않는 경향이 있습니다. 콘텐츠 탐색이 cta이기 때문인데요.

overflow menu의 trigger는 chip이지만 menulist는 시트/드롭다운/셀렉트박스/모달이 될 수 있어야 합니다. 모달이 가장 범용적. sorting/filtering 등 역할 기반 컴포넌트를 만들고 chip을 mixin.

Best practices

  • 아티클 레벨에서 사용합니다. 섹션 레벨과 페이지 레벨에서는 충분한 여백으로 콘텐츠를 구획합니다.

    Callouts

    시스템이나 브라우저의 상태 이상, 사용자의 행동에 피드백합니다.

    a.k.a. Alert Message Callout Banner Information

    Best practices

    • 모든 액션에 피드백할 필요는 없습니다. (ex bad case: 인스타그램 앱에서 로그인되었다는 toast. 홈으로 이동과 피드 노출만으로 로그인 상태를 인지할 수 있다.)
    • 닫기나 되돌리기와 같은 액션을 포함하지 않습니다. 인터렉션이 필요한 경우에는 주목도가 높은 모달 형태를 사용하세요.
    • 사용자의 귀책이 아닌 경우, 단순 "오류" 대신 "시스템 오류"와 같은 용어를 사용하여 사용자가 잘못한 것처럼 느끼지 않도록 합니다.
    • Don't show more than one at a time.
    • 사라지는 시간을 정의.
