Buttons

과업 수행을 위한 페이지 이동, 데이터 제출 등 사용자 여정의 핵심적인 액션입니다.

  • 버튼은 페이지 내에서 여러 번 반복될 수 없는 최종적인 인터렉션입니다. 단, 전송이 실패하는 경우에는 여러 번 반복해서 실행할 수 있습니다.
  • Feedback needed. 실행 후 피드백합니다. 물리적인 인터페이스의 출현 뿐 아니라 예측할 수 있는 페이지 이동이나 페이지 새로고침, 정보 구조의 변화도 피드백으로 취급될 수 있습니다. (bad case. 인스타그램의 로그인 성공 toast)
  • 레이블은 사용자가 다음에 일어날 일을 예측할 수 있도록 작성하세요. 텍스트 레이블을 생략하는 경우 관습적인 메타포의 아이콘(북마크/하트/카트)을 사용하고 대체 텍스트와 툴팁을 제공해야 합니다. 장식적인 용도로 아이콘을 사용하지 마세요.
  • 형태와 레이블로 버튼으로 기능해야 합니다. 툴팁으로 행동을 유도하거나, 보조 텍스트로 실행을 유도하거나 실행 결과를 가이드할 필요가 없도록 합니다. (이 시스템에는 정의하지 않지만 세로 정렬되는 버튼 그룹에서 text button이 아래에 배치되는 패턴을 고려하여 버튼의 아래 가이드 텍스트를 배치하지 않습니다.)
  • memo: secondary style과 large size의 사용 빈도 낮음. 현재 large 버튼은 프로모션 페이지의 call to action으로 제한적으로 사용합니다.
